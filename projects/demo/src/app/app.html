Queue

A burst is not painted all at once. Each toast enters a queue and a consumer releases them one at a time as slots free up (max), spaced by stagger ms. Nothing is lost: the extras appear as visible toasts expire. 

provideSnackng({
  max: 4,                    // visible at once
  overflow: 'queue',         // or 'dismiss-oldest' so the newest wins
  stagger: 90,               // ms between releases; 0 = all at once
})

Animation

Slides in from the edge it is anchored to (400ms) and out in 200ms. Under prefers-reduced-motion: reduce the travel collapses to 0 and only the fade remains — the library never overrides that system setting.

Your system requests reduced motion: {{ prefersReducedMotion() ? 'yes → fade only' : 'no → you will see the slide' }}

Theming

Recolour everything from your own CSS with variables, no ::ng-deep:

:root {
  --snackng-success-bg: linear-gradient(135deg, #059669, #064e3b);
  --snackng-radius: 8px;
}
